# Contributing to AgroPulse AI

First off, thank you for considering contributing to AgroPulse AI! It's people like you that make AgroPulse AI such a great tool for farmers.

## Code of Conduct

This project and everyone participating in it is governed by the [AgroPulse AI Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report for AgroPulse AI. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Check if the bug has already been reported** by searching on GitHub under Issues.
- **Use the Bug Report template** to create a new issue.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion for AgroPulse AI, including completely new features and minor improvements to existing functionality.

- **Check if the enhancement has already been suggested** by searching on GitHub under Issues.
- **Use the Feature Request template** to create a new issue.

### Pull Requests

- Fill in the [required template](.github/PULL_REQUEST_TEMPLATE.md).
- Do not include any sensitive information (API keys, credentials) in your code.
- Ensure the CI tests pass.
- Follow the existing code style.

## Styleguides

### Git Commit Messages

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line

### JavaScript Styleguide

- We use ESLint for code linting. Run `npm run lint` before committing.
- Use functional components and hooks for React.
- Keep components small and focused.
